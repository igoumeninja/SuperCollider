Description of the OSC message format with which the structure of the piece Stasis is sent to all computers. 


There are two kinds of OSC messages: '/sync' and 's_branch' or 'l_branch'. Here are first some examples taken from the beginning of a piece:


Example 1: /sync, s_start_, 0, 4181, 0


The meaning of each part of this OSC message is: 


/sync is the message for SyncAction.

s_start_ shows that:

- this is a tree with short branches first (s),

- this is the start of a branch (start) 

- this is the root of the tree (level 0, whole piece): no branch name is present (no A or B letters)

0 shows that this is beat number 0

4181 shows that there are 4181 beats in this branch (4181 beats in the whole piece)

0 shows that the level of this branch is 0 (meaning, this is the root of the tree)

 

Example 2: /sync, s_start_A, 0, 1597, 1


/sync is the message for SyncAction.

s_start_A shows that:

- this is a tree with short branches first (s),

- this is the start of a branch (start) 

- this is is branch A (level 1, first branch)

0 shows that this is beat number 0

1597 shows that there are 4181 beats in this branch (1597 beats in the first branch of the piece)

1 shows that the level of this branch is 1 (first level of branches in the tree)


Example 3: s_branch, 2584, 5, 8, 4181, 3.5, 19.909523010254, 7.6047620773315


s_branch is the OSC. It is sent for the GUI. It is sent at each start of branch. 

It sends the s_start data plus some more information, as follows:

2584 shows that this is beat number 2584

5 the number of beats in this branch 

8 the level of this branch

4181 the total size of the piece in beats

3.5 the current playback tempo

19.90.. the total duration of the piece, in minutes

7.60.. the duration left until the end of the piece, in minutes


Example 4: /sync, beats, 1, 3.5, 4, 0, 0, freq, amp, pan, trig


This message is sent every single beat by the SyncSender. It is sent independently of the Fibonacci Tree. It is sent by the sender for any beat of any structure that it plays. 


/sync is the message for SyncAction.

beats this means that this is one of the beats of the piece.

The Stasis player sends every single beat of the piece with the message

combination /sync beats. 

1 shows that this is beat number 1, from the start of playing of the sender, 

not from the start of the piece.

3.5 the current playback tempo of the clock

4 number of beats per bar in the clock setting

0 baseBarBeat (this is a clock property not used in the present project)

0 baseBar (this is a clock property not used in the present project)

freq, amp, pan, trig Additional arguments that can have numeric values if the event of the pattern played 

has appropriate objects for these keys. Not used in the present project.



Example 5: /sync, s_end_BBAAAAAAA, 2586


/sync is the message for SyncAction.

s_end_BBAAAAAAA Signifies the end of a branch in the Fibonacci tree. Details: 

- s signifies that this is a tree with short branches first.

- end signifies that this is the end of a branch.  

- this is the branch corresponding to branching BBAAAAAAA

2586 shows that this is beat number 2584

===== Message format:


| type of message           | format of message          | argument 1                      |

| branch                    | /[s,l]_[A,B]*_[start|end]  | length of branch  in beats      |

| leaf                      | /leaf                      | value of the leaf               |

| beat                      | /beat                      | beat count since start of piece |


Explanation: 


- "s" and "l" indicate the direction of the generation of the tree as follows: 


- "l" if the branches are ordered with the longer one first, (A2 or B1 above)

- "s" if they are ordered with the shorter one first (A1 or B2 above).


- "A" and "B" indicate the branch of the tree as follows: 


- A indicates the first branch in a node of the tree and B the second one. 

- A's and B's are repeated to specify subbranches, For example: 

AB = Branch B of branch A

BAA = Branch A of branch A of branch B. 

Following diagram shows how labels indicate branches in a tree of 3 branch levels: 


Level 1: A                                             B

Level 2: AA                        AB                  BA         BB

Level 3: AAA           AAB         ABA        ABB      BAA    BAB BBA  BBB


- "start" and "end" indicate the start of a branch or end of a branch. 




===== Structure of the tree and its variants


There are four variants: 


The variant pair A1 and A2 is created by incrementing the values in the 'current' branch by 1 at each iteration: 


A1. Ascending leaf numbers, smaller branches first: 


generated by: next = [prev, current + 1]; prev = current;


[ [ 1, [ 2, 3 ] ], [ [ 2, 3 ], [ 3, [ 4, 5 ] ] ] ]


A2. Descending leaf numbers, larger branches first: 


generated by: next = [current + 1, prev]; prev = current;


[ [ [ [ 5, 4 ], 3 ], [ 3, 2 ] ], [ [ 3, 2 ], 1 ] ]


The variant pair B1 and B2 is created by incrementing the values in the 'current' branch

*and* the values in the 'prev' branch by 1 at each iteration: 


B1. Ascending leaf numbers, larger branches first: 


generated by: next = [current, prev + 1]; prev = current + 1;


[ [ [ [ 1, 2 ], 3 ], [ 3, 4 ] ], [ [ 3, 4 ], 5 ] ]


B2. Descending leaf numbers, smaller branches first: 


generated by: next = [prev + 1, current]; prev = current + 1;


[ [ 5, [ 4, 3 ] ], [ [ 4, 3 ], [ 3, [ 2, 1 ] ] ] ]
